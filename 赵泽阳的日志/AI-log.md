# AI-Log（对话形式）
## 交互1：搭建虚拟环境阶段
**用户**：PyCharm社区版中文界面，想创建venv虚拟环境，还遇到了解释器关联不上的问题，该怎么解决？
**AI**：你可以按中文界面的步骤操作：第一步打开PyCharm，点击底部「终端」，进入项目根目录后执行`python -m venv venv`创建虚拟环境；第二步点击右下角「Python 3.13」→「添加解释器」，选择「Virtualenv 环境」→「已有环境」，找到项目内`.venv\Scripts\python.exe`关联；如果终端提示“python不是内部命令”，需要去电脑「设置」→「环境变量」→「Path」中添加Python安装路径和Scripts路径。

## 交互2：依赖安装阶段
**用户**：虚拟环境里安装了Flask相关依赖，但PyCharm还是提示“未解析的引用”，代码标红该怎么处理？
**AI**：首先确认终端开头显示「(venv)」（激活了虚拟环境），然后执行带版本号的安装命令：`pip install flask==2.3.3 flask-sqlalchemy==3.1.1 flask-login==0.6.3 werkzeug==3.1.3`；安装完成后点击PyCharm顶部「文件」→「使缓存失效并重启」，重启后标红会自动消失，这样能避免依赖版本冲突和安装路径错误的问题。

## 交互3：模板报错阶段
**用户**：浏览器显示jinja2.exceptions.TemplateNotFound: login.html，该怎么解决？
**AI**：这个错误是Flask找不到模板文件导致的，你需要先确认项目根目录下有「templates」文件夹（必须是小写且带s，不能写成template）；然后把login.html、register.html等文件都放进这个文件夹，文件名要全小写（不能是Login.html）；如果文件路径没问题，重启后端后再访问，就能正常加载模板了，也可以在VSCode里直接拖拽文件调整路径。

## 交互5：数据库报错阶段
**用户**：运行后提示sqlalchemy.exc.OperationalError: no such column: post.likes，该怎么解决？
**AI**：这个错误是旧数据库没有新增的likes字段导致的，最快捷的解决方法是：先停止PyCharm中运行的后端，然后找到项目文件夹里的「social.db」文件并删除，再重新点击运行「app.py」，代码里的`db.create_all()`会自动重建包含likes字段和新表的数据库，实训场景下无需备份旧数据，这样操作最高效。
